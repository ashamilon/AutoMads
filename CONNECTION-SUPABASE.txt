WHY ERRORS HAPPEN
-----------------
1) "Can't reach database" on db.XXXX.supabase.co → your network is IPv4-only; use POOLER URLs below.
2) "Tenant or user not found" → the wrong AWS region in the host (aws-0-WRONG.pooler...) — must match YOUR project.

WHAT TO DO (copy from Supabase — no typing)
---------------------------------------------
1. Open this link (same project):
   https://supabase.com/dashboard/project/buhkwdutjdvgpwlqlhaf

2. Click the green "Connect" button (top of the project).

3. In the connection dialog:
   - Pick "ORM" or "Prisma" if you see it, OR choose "Transaction pooler" and "Session pooler".

4. Copy the URI that shows PORT 6543 (transaction / pooler).
   - Paste it as DATABASE_URL in your .env file.
   - If it does not already contain "?pgbouncer=true", add at the end:
     ?pgbouncer=true&connection_limit=1&sslmode=require
     (If it already has "?", use "&pgbouncer=true&connection_limit=1&sslmode=require" instead.)

5. Copy the URI that shows PORT 5432 (session pooler).
   - Paste it as DIRECT_URL in your .env file.
   - Add at the end if missing: ?sslmode=require

6. In BOTH strings, replace the password placeholder with YOUR database password.
   - If your password contains ! characters, change each ! to %21 in the URL only.

7. Save .env, then double-click sync-database.bat again.

IMPORTANT
---------
- User in the URI must look like: postgres.buhkwdutjdvgpwlqlhaf (with a dot), NOT only "postgres".
- Host must start with aws-0- and end with .pooler.supabase.com — copy from Supabase exactly.
